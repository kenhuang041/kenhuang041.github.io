🏰 dp[i][j][s] 三維陣列完全解析

dp[i][j][s] = 前i行棋盤中，第i行使用第j種狀態，總共放了s個國王的方案數

🔢 第一維：i (行數)

意義：考慮前i行棋盤

範圍：1 到 總行數

例子：i=3 表示考慮前3行

📋 第二維：j (狀態編號)

意義：第i行使用的狀態編號

範圍：1 到 cnt (總狀態數)

例子：j=3 表示使用狀態3(.K.)

👑 第三維：s (國王數量)

意義：總共放置的國王數量

範圍：0 到 k (目標數量)

例子：s=2 表示總共2個國王

🎯 具體例子：3×2棋盤，放置2個國王

  • 棋盤寬度 n = 3
  • 棋盤高度 rows = 2
  • 目標國王數 k = 2

📊 DP陣列初始化 (第一行)

初始化規則：
for (int j = 1; j <= cnt; j++) {
    if (now[j] <= k) {
        dp[1][j][now[j]] = 1;
    }
}
DP值 意義 說明
dp[1][1][0] = 1 第1行用狀態1，放0個國王 第1行是空的，有1種方案
dp[1][2][1] = 1 第1行用狀態2，放1個國王 第1行是..K，有1種方案
dp[1][3][1] = 1 第1行用狀態3，放1個國王 第1行是.K.，有1種方案
dp[1][4][1] = 1 第1行用狀態4，放1個國王 第1行是K..，有1種方案
dp[1][5][2] = 1 第1行用狀態5，放2個國王 第1行是K.K，有1種方案

🔄 DP轉移過程 (第二行)

轉移公式：
dp[i][j][s] += dp[i-1][prev][s - now[j]]

條件：第(i-1)行的狀態prev和第i行的狀態j不能衝突

🎯 最終答案計算

求解目標：恰好放置k=2個國王的方案數
答案 = dp[2][1][2] + dp[2][2][2] + dp[2][3][2] + dp[2][4][2] + dp[2][5][2]
DP值 對應配置 方案數
dp[2][1][2] 第2行用狀態1(...), 總共2個國王 1 (第1行必須用狀態5)
dp[2][2][2] 第2行用狀態2(..K), 總共2個國王 1 (第1行用狀態4)
dp[2][3][2] 第2行用狀態3(.K.), 總共2個國王 1 (第1行用狀態4)
dp[2][4][2] 第2行用狀態4(K..), 總共2個國王 1 (第1行用狀態2)
dp[2][5][2] 第2行用狀態5(K.K), 總共2個國王 0 (第1行無法提供0個國王)
總答案 = 1 + 1 + 1 + 1 + 0 = 4種方案

💡 關鍵理解要點

🔢 第一維 i

從上到下逐行構建

i=1: 只考慮第1行

i=2: 考慮第1-2行

i=3: 考慮第1-3行

📋 第二維 j

當前行使用的狀態

決定了當前行國王的具體位置

必須與上一行的狀態相容

👑 第三維 s

累計的國王總數

從0開始逐漸增加

最終要達到目標數量k
