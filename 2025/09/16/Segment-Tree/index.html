    線段樹

    發表於 2025-09-16 分類於 資料結構
    快速處理區間相關內容的資料結構

    求區間和跟修改都優於普通陣列的資料結構
    複雜度跟BIT一樣，都是O(log n)
    但更直觀一點? (我是這麼覺得^^)

    結構

    線段樹是一個完整二元樹，假設是一個長度為9的線段樹，則其二元樹如下圖所示：

    用陣列存取，若根節點為tree[1]（通常從1開始），則其左右子節點存放位置如下

    • 左子節點 -> 2i
    • 右子節點 -> 2i+1

    依照規則填入數字，所以tree[1] = [0,8], tree[2] = [0,4], tree[3] = [5,8]

    接下來就可以用程式碼實現了

    功能

    查詢區間和方法如下：

    • 從根節點開始向下找，若當前節點 ql <= l && r <= qr，則代表此節點被包含在查詢區間內
    • 如果節點完全在查詢範圍外（qr < l || r < ql），可以直接終止遞迴
    • 否則繼續遞迴查詢左右子樹，並將結果相加

    修改單點也是差不多的方式，從根節點向下找到需更改的葉節點，再透過遞迴的回溯更新被影響的節點就好了


    總結

    線段樹是一個非常實用的資料結構，特別適合處理區間查詢和單點修改的問題。雖然實作稍微複雜，但一旦理解其原理，就能靈活運用於各種競程題目中。

    時間複雜度

    • 建構：O(n)
    • 查詢：O(log n)
    • 修改：O(log n)

    空間複雜度

    • O(4n)（通常開 4 倍大小的陣列）
